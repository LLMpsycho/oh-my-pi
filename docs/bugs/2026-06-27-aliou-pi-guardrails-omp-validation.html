@aliou/pi-guardrails fails OMP extension validation

Summary

Installing @aliou/pi-guardrails into OMP fails during extension validation. The package is an upstream Pi extension and its settings UI imports the upstream Pi TUI runtime package @earendil-works/pi-tui, which is not present in the OMP plugin workspace.

Date observed

2026-06-27

Affected package

Package
@aliou/pi-guardrails
Attempted versions
latest and 0.14.0
Reason for pinning 0.14.0
The latest release was inside the configured npm minimum-release-age quarantine window. Version 0.14.0 was the newest release-age-eligible version found during the install attempt.

Reproduction

omp plugin install @aliou/pi-guardrails
omp plugin install @aliou/pi-guardrails@0.14.0

Observed failure

Plugin @aliou/pi-guardrails extension validation failed:
/Users/besi/.omp/plugins/node_modules/@aliou/pi-guardrails/extensions/path-access/index.ts: ResolveMessage: Cannot find module '@earendil-works/pi-tui' from '/Users/besi/.omp/plugins/node_modules/@aliou/pi-utils-settings/src/components/array-editor.ts'
/Users/besi/.omp/plugins/node_modules/@aliou/pi-guardrails/extensions/guardrails/index.ts: 10 errors building "/Users/besi/.omp/plugins/node_modules/@aliou/pi-guardrails/package.json"
/Users/besi/.omp/plugins/node_modules/@aliou/pi-guardrails/extensions/permission-gate/index.ts: 10 errors building "/Users/besi/.omp/plugins/node_modules/@aliou/pi-guardrails/package.json"

Problematic extension entries

  • extensions/path-access/index.ts
  • extensions/guardrails/index.ts
  • extensions/permission-gate/index.ts

Root cause

@aliou/pi-guardrails depends on @aliou/pi-utils-settings. During OMP extension validation, @aliou/pi-utils-settings/src/components/array-editor.ts imports @earendil-works/pi-tui. That package is an upstream Pi runtime/UI peer, not an OMP-native plugin dependency.

Installing @earendil-works/pi-tui as an OMP plugin just to satisfy validation is not a safe fix. It would mix upstream Pi runtime packages into the OMP plugin workspace and could mask an API compatibility problem.

Impact

  • @aliou/pi-guardrails is not installed.
  • OMP rolls back the failed install rather than leaving a partial package installed.
  • The failure does not affect already-installed healthy plugins.

Current plugin health after rollback

After the failed install attempts, omp plugin doctor reported 7 ok, 3 warnings, and 0 errors. The warnings were for pre-existing package entries without omp or pi manifests: compound-engineering, context7, and superpowers.

Healthy extension/plugin entries included omp-discord-presence, pi-continuous-learning, pi-generative-ui, and pi-lens.

Resolution

Fixed in OMP by extending the legacy Pi extension validation graph to include plugin-local Pi helper packages such as @aliou/pi-utils-settings. Their source files now receive the same legacy Pi import rewrites as the extension entry, so imports like @earendil-works/pi-tui resolve through OMP's compatibility shim during validation.

The graph expansion is intentionally limited to plugin-local helper packages whose package basename starts with pi-, so general third-party dependencies are not pulled into the rewrite hook.

Post-fix verification

  1. Keep the install-level regression test in packages/coding-agent/test/plugin-install-validation.test.ts passing. It exercises PluginManager.install("@aliou/pi-guardrails@0.14.0") through extension validation.
  2. Before installing the real plugin into a persistent OMP home, run a sandbox install with the patched loader and confirm omp plugin doctor reports no new errors.
  3. Do not force-install upstream Pi runtime peers into OMP. The compatibility loader should own the @earendil-works/pi-tui remapping path.
